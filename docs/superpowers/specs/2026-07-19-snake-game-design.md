# 贪吃蛇 Web 小游戏 — 设计文档

## 背景与目标

在 `chrys-data-platform-demo` 仓库中新增一个独立的 Web 小游戏子目录,作为后续"小游戏合集"的起点。本次只实现**贪吃蛇**一个游戏,目的是跑通整条链路(目录结构、Vue + Vite 工程化、逻辑/渲染分离、单元测试),确认后再逐步补齐 2048、扫雷等。

小游戏合集与主项目(`src/` 下的 Python 任务看板)完全隔离:技术栈不同、依赖不同、运行方式不同,仅共享同一个 Git 仓库。主项目的"零第三方运行时依赖"原则不适用于 `games/` 子目录。

## 范围

**本次只做:**
- `games/` 子目录的 Vue 3 + Vite 工程骨架
- 合集首页(一张贪吃蛇入口卡片,预留后续游戏的卡片位)
- 贪吃蛇游戏的完整实现(逻辑 + 渲染 + 键盘控制)
- 贪吃蛇逻辑的单元测试(Vitest)

**本次不做:**
- 2048、扫雷等其他游戏(仅预留目录与路由扩展点)
- 排行榜、存档、音效、设置面板、主题切换
- 数据上报/归因埋点(Chrys hook 已在设备层面监控,无需主动接入)

## 贪吃蛇游戏规格

| 维度 | 说明 |
|---|---|
| 棋盘 | 正方形网格,默认 20×20;边界即墙,撞墙判负 |
| 移动 | 蛇按当前方向每 tick 前进一格;ticked 由组件的定时器驱动(默认约 150ms/格) |
| 食物 | 棋盘随机一格(避开蛇身)生成一个食物;吃到后蛇身 +1,分数 +10 |
| 操作 | 方向键或 WASD;**禁止 180° 反向**(直接掉头判负);同一 tick 内仅以最后一次有效方向为准 |
| 胜负 | 撞墙、撞自身、或 180° 反向 → 游戏结束(无"胜利"终态,比分数) |
| 局制 | 单局制;结束后可"重开一局",重置棋盘与分数 |

### 蛇身与坐标模型

- 蛇身用一个**有序坐标列表**表示,`snake[0]` 为蛇头,其余为身体。移动 = 在头前插入新坐标 + 尾部弹出(吃食物时不弹尾部)。
- 方向用单位向量表示:`{row, col}` 取值于 `{(-1,0),(1,0),(0,-1),(0,1)}`。
- 所有边界与碰撞判定基于网格坐标,不依赖像素。

## 架构:逻辑与渲染分离

这是本设计的核心约束,也是仓库"重视测试"基因的延续:

- **`engine.js`**:框架无关的纯函数集合。输入是"当前状态 + 一个动作",输出是"新状态"。不引用 DOM、不引用 Vue、不产生副作用。状态包括网格尺寸、蛇身、方向、食物位置、分数、游戏是否结束。
- **`index.vue`**:只负责把 engine 状态渲染到 `<canvas>`,把键盘事件转成动作传给 engine,并用定时器驱动 tick。
- **`engine.test.js`**:直接对纯函数断言,覆盖移动、吃食物生长、撞墙、撞自身、180° 反向、食物不生成在蛇身上等场景。

> 采用不可变更新(engine 返回新状态对象而非原地 mutate),使测试可预测、渲染层易于做"状态快照 → 重绘"。

## 技术栈

- **Vue 3**(`<script setup>` 组合式 API)
- **Vite**(开发服务器 + 构建)
- **Vue Router**(首页 ↔ 贪吃蛇页面切换)
- **Vitest**(engine 纯逻辑单元测试)
- 渲染:`<canvas>`(贪吃蛇重绘频繁,canvas 比 DOM 合适)

> 选 Vue 3 + Vite 而非原生 JS:组件化便于后续扩展更多游戏,工程化(热更新、构建、测试)成本可接受。Node 环境需求由 `games/README.md` 说明。

## 与主项目的关系

- 完全隔离:不修改主项目任何文件
- `games/` 自带 `.gitignore`(忽略 `node_modules/`、`dist/`)
- 需确认主仓库根 `.gitignore` 是否已忽略 `node_modules`、`dist`;若未忽略,本次补充(仅这两条,且写成对 games 子树友好的形式)

## 验收标准

1. 在 `games/` 下执行 `npm install && npm run dev` 后,浏览器可打开首页,点击进入贪吃蛇并可正常游玩
2. 方向键/WASD 可控;吃食物长身、加分;撞墙/撞自身/180° 反向正确判负;可重开
3. `npm run test` 通过,engine 逻辑测试全绿
4. `npm run build` 成功产出 `dist/`
5. 主项目 `python -m unittest discover -s tests -v` 仍然全绿(未被影响)

## 后续扩展(非本次范围)

新增游戏只需:在 `games/src/games/` 下建子目录(`index.vue` + `engine.js` + `engine.test.js`)、在 `router.js` 加一条路由、在 `App.vue` 加一张卡片。`GameShell.vue` 可复用作为统一外壳。
